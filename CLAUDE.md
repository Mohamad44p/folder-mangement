# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project shape

A Next.js 16 / React 19 **frontend-only** folder-management UI, originally generated by v0.app and kept in sync with deployments. There is no backend, no API routes — all state is persisted to `localStorage`. Every meaningful component is a client component (`"use client"`).

## Commands

Package manager is **bun** (`bun.lock` is committed; `pnpm-lock.yaml` was removed).

- `bun install` — install deps
- `bun run dev` — Next dev server
- `bun run build` — production build
- `bun run start` — serve the built app
- `bun run lint` — `eslint .`

There is no test runner configured. `next.config.mjs` sets `typescript.ignoreBuildErrors: true`, so `bun run build` will not catch type errors — run `bunx tsc --noEmit` to type-check explicitly.

## Architecture

### Provider stack (`app/layout.tsx`)
The app is wrapped in a fixed provider order — keep it intact when editing layout:
`SettingsProvider` → `I18nProvider` → `GenerationProvider` → `DndProvider` → `FolderProvider`. `I18nProvider` reads `language` from `SettingsProvider`, and several lower providers read folder state, so reordering will break hooks at runtime.

### Central state: `contexts/folder-context.tsx`
One large context owns folders, files, smart folders, saved searches, workspaces, trash, lightbox/slideshow/compare modal state, drag selection, sidebar/trash open flags, and dozens of CRUD operations. New folder/file features should attach here rather than introducing a sibling context. Folders are flat in storage but form a tree via `parentId`; helpers like `getDescendantIds`, `buildPathTitles`, `getFolderTree` traverse it.

### Persistence (`lib/folder-storage.ts`, `contexts/settings-context.tsx`)
- Folders: `localStorage["folder-mgr:v2"]` as `{ folders, version: 1 }`.
- Settings (theme/language/accent/density/reduceMotion): `localStorage["folder-mgr:settings:v2"]`.
- Uploaded files are read via `FileReader.readAsDataURL` and stored inline as data URLs — quota errors are swallowed silently. Be conservative about what gets persisted; nothing else clears storage on quota failure.

### i18n (`contexts/i18n-context.tsx`, `lib/i18n-dict.ts`, `lib/localize.ts`)
- Two languages: `en` and `ar`. Arabic flips `<html dir="rtl">` and uses Cairo font (set in `app/layout.tsx`).
- Static UI strings: add a key to **both** `en` and `ar` blocks in `lib/i18n-dict.ts`, then use `const { t } = useT()`.
- Dynamic content (folder titles, tags, dates, byte sizes, numerals) goes through `lib/localize.ts` — `localizeTitle`, `localizeTag`, `formatDateLocalized`, `formatBytesLocalized`, `localizeNumber` (Arabic-Indic digits). Don't format these inline.

### Drag-and-drop (`contexts/dnd-context.tsx`)
Custom HTML5 DnD — not `react-dnd` or `dnd-kit`. Use `useDraggable({ kind, ... })` and `useDropTarget({ id, accept, onDropItem, onDropFiles, canDrop })`. The `"os-files"` accept type handles drops from the operating system.

### Generation animation (`contexts/generation-context.tsx`)
Runs the per-card progress/sparkles/reveal animation. Card creation in `app/page.tsx` calls `startGeneration(id, onComplete)`; the callback flips `isGenerating` off via `setFolderGenerating`. Default duration is 10 s.

### Theming and CSS
- Tailwind v4 via `@tailwindcss/postcss` (no `tailwind.config.*`); design tokens live in `app/globals.css`.
- shadcn config: `components.json` style `new-york`, base color `neutral`, `lucide` icons. Only two shadcn primitives are vendored (`components/ui/dropdown-menu.tsx`, `select.tsx`); the rest of the UI is hand-rolled. When adding a shadcn component, drop it into `components/ui/`.
- User accent color writes `--accent-user` / `--accent-user-soft` / `--accent-user-ring` on `:root`. **Do not write to `--accent`** — that's shadcn's token and we deliberately don't clobber it.
- Density and reduce-motion are exposed as `data-density` and `data-reduce-motion` attributes on `<html>`; `MotionConfig` wires Framer Motion to the same flag.

### Path alias
`@/*` → repo root (configured in `tsconfig.json` and shadcn aliases). Imports like `@/components/...`, `@/lib/...`, `@/contexts/...` are the convention.

### Single-file-playing invariant (`lib/play-state-manager.ts`)
A module-level singleton ensures only one previewable file plays across the app. Any new audio/video preview component should call `setPlayingFile(id)` on play and subscribe via `subscribeToPlayState` to pause itself when another file starts.

### Smart folders (`lib/smart-folder-engine.ts`)
Rule-based virtual folders evaluated client-side over the in-memory file set; supported fields are `tag | type | favorite | name | size | uploaded`. Add new rule fields here and in the editor (`components/smart-folder-editor.tsx`).

## v0.app sync

The repo auto-syncs from v0.app deployments (see `README.md`). Treat changes as potentially overwritten by v0 if the user is also editing through that tool — prefer surgical edits and avoid sweeping reformats.
